# 2025A_FE-A_33 %% ex. 2025A_FE-S_83 %%

In a system with the configuration and communication services shown in the figure below, which of the following is the most appropriate location for the installation of a WAF as a countermeasure for a vulnerability in a web application? Here, the WAF has no function to encrypt or decrypt communications.

[2025A_FE-A_33.png]

a) A
b) B
c) C
d) D
?
b) B
### Step 1: Understand what a WAF does

A **WAF (Web Application Firewall)** inspects HTTP/HTTPS traffic between clients and a web server to block malicious requests (SQL injection, XSS, etc.). Since this WAF **cannot encrypt or decrypt**, it must be placed where traffic is in **plaintext (HTTP)** — i.e., between the SSL termination point and the web server.

### Step 2: Analyze the network diagram

In the typical configuration shown:
- **A** — Before the firewall / internet edge (traffic is still encrypted HTTPS)
- **B** — Between the reverse proxy/load balancer (SSL terminator) and the web server (traffic is plaintext HTTP here)
- **C** — Between the web server and the database
- **D** — Inside the server's internal network, after the web application

### Step 3: Select the correct location

Since the WAF cannot decrypt, it must see plaintext traffic. After SSL termination (e.g., at a reverse proxy), traffic flows as HTTP to the web server. **Position B** is between the SSL terminator and the web server, where the WAF can inspect unencrypted HTTP requests.

### Answer: b) B
