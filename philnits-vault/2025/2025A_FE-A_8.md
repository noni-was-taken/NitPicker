# 2025A_FE-A_8
There exists a main process and the two (2) interrupt processes A and B, as shown in the table below. Multiplex interrupts are permitted. When interrupts A and B occur with the timing shown in the diagram below, how many milliseconds (ms) can the main process use the CPU during the time period between 0 ms and 5 ms? Here, the overhead from calling an interrupt process and resuming from it can be ignored.

[2025A_FE-A_8.png]

| Interrupt | Processing time | Priority |
|-----------|----------------|----------|
| A         | 0.5 ms          | High     |
| B         | 1.5 ms          | Low      |

a) 2
b) 2.5
c) 3.5
d) 5
?
b) 2.5
### Step 1: Set up the timeline (0 ms to 5 ms)

From the timing diagram (standard version of this problem):
- Interrupt B occurs at 1 ms (low priority, 1.5 ms processing)
- Interrupt A occurs at 1.5 ms (high priority, 0.5 ms processing — preempts B)

### Step 2: Trace CPU usage

- **0–1 ms**: Main process runs → **1 ms**
- **1 ms**: Interrupt B starts (low priority)
- **1.5 ms**: Interrupt A preempts B (high priority)
- **1.5–2 ms**: Interrupt A runs (0.5 ms) → completes at 2 ms
- **2–3.5 ms**: Interrupt B resumes and completes (1.5 ms − 0 ms already done = 1.5 ms) → done at 3.5 ms
- **3.5–5 ms**: Main process resumes → **1.5 ms**

### Step 3: Total main process CPU time

$$1 \text{ ms} + 1.5 \text{ ms} = 2.5 \text{ ms}$$

### Answer: b) 2.5
