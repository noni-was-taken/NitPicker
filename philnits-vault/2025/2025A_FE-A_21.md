# 2025A_FE-A_21
Which of the following is the clause that is inserted into blank A of the SQL statement in order to display the names of employees who work in the same department as any employee located in 'New York'?

```
Employees (employee_id, employee_name, salary, department_id)
Departments (department_id, department_name, location)
```

```sql
SELECT employee_name FROM Employees WHERE [A]
```

a) `department_id IN (SELECT department_id FROM Departments WHERE location = 'New York');`
b) `department_id NOT IN (SELECT department_id FROM Departments WHERE location = 'New York');`
c) `department_id IN (SELECT department_id FROM Employees WHERE location = 'New York');`
d) `department_id = ALL (SELECT department_id FROM Employees WHERE location = 'New York');`
?
a) `department_id IN (SELECT department_id FROM Departments WHERE location = 'New York');`
### Explanation

The goal is to find employees whose `department_id` matches **any** department located in 'New York'. The correct approach:
1. Use a subquery on the **Departments** table (which holds `location`) to get `department_id`s in New York
2. Use `IN` to match employees with those department IDs

- Option b) uses `NOT IN`, which would return employees *not* in New York departments
- Option c) queries `location` from the **Employees** table, but `location` is a column of **Departments**, not Employees
- Option d) uses `= ALL`, which would require the department_id to match *all* returned IDs — logically incorrect for this use case
