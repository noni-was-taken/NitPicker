# 2025A_FE-A_19 %% ex. 2025A_FE-A_83 %%

When the functional dependencies shown below are satisfied, which of the following is the transitive functional dependency that holds?

- {OrderCode, ProductCode} → {CustomerOrderQuantity, OrderAmount}
- OrderCode → {OrderDate, CustomerCode, OrderManagerCode}
- ProductCode → {ProductName, SupplierCode, ProductSalePrice}
- SupplierCode → {SupplierName, SupplierAddress, SupplierManagerCode}
- CustomerCode → {CustomerName, CustomerAddress}

a) OrderCode → CustomerCode → CustomerAddress
b) OrderCode → ProductCode → CustomerOrderQuantity
c) ProductCode → SupplierCode → ProductSalePrice
d) SupplierCode → SupplierManagerCode → SupplierAddress
?
a) OrderCode → CustomerCode → CustomerAddress
### Explanation

A **transitive functional dependency** exists when $A \to B$ and $B \to C$, giving $A \to C$ transitively.

Evaluating option a):
- $\text{OrderCode} \to \text{CustomerCode}$ ✅ (given)
- $\text{CustomerCode} \to \text{CustomerAddress}$ ✅ (given)
- Therefore: $\text{OrderCode} \to \text{CustomerAddress}$ transitively ✅

Evaluating others:
- b) OrderCode does not determine ProductCode (they are both part of a composite key)
- c) ProductCode → SupplierCode is valid, but SupplierCode → **ProductSalePrice** is not given; ProductSalePrice is determined by ProductCode directly
- d) SupplierManagerCode is a *dependent* of SupplierCode, not a determinant of SupplierAddress
