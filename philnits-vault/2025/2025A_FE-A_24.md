# 2025A_FE-A_24
PC5 has been newly added to the network. It wants to send data to PC4 using SSH protocol. It must resolve the MAC address of PC4 from its IP address before sending the data. Which of the following is the destination MAC address that PC5 will use in ARP request?

[2025A_FE-A_24.png]

a) 00:00:00:00:00:00
b) 78:E1:00:3C:A1:8B
c) 78:E1:00:3C:A1:9F
d) FF:FF:FF:FF:FF:FF
?
d) FF:FF:FF:FF:FF:FF
### Step 1: Understand ARP requests

ARP (Address Resolution Protocol) is used to discover the MAC address of a host given its IP address on a local network.

### Step 2: ARP request destination

An ARP request is a **broadcast** message sent to all devices on the local network segment. The destination MAC address for ANY ARP request frame is always the broadcast address:

$$\text{FF:FF:FF:FF:FF:FF}$$

This ensures every device on the LAN receives the request. Only the device with the matching IP address replies with its MAC address (unicast ARP reply).

### Step 3: Why not the other options?

- `00:00:00:00:00:00` — This is a null / unset MAC, not used for ARP requests.
- `78:E1:00:3C:A1:8B` / `78:E1:00:3C:A1:9F` — These are specific unicast MACs (router/host); ARP doesn't know the target's MAC yet (that's the whole point).

### Answer: d) FF:FF:FF:FF:FF:FF
